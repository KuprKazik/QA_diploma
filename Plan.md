# 1.Перечень автоматизированных сценариев

1) Работа кнопок "Купить" и "Купить в кредит" (кнопки нажимаются и происходит переход к банковскому сервису)
2) Переход на соответствующий банковский сервис (с кнопки "Купить" - к Payment Gate, с кнопки "Купить в кредит" - к Credit Gate)
3) В Payment/Credit Gate проверка суммы транзакции (чтобы совпадала с суммой, предложенной в предзаказе/на главной странице)
4) Валидация номера дебетовой/кредитной карты
5) Валидация даты выпуска карты
6) Валидация СVV-кода (3 знака)
7) Работа кнопки подтверждения оплаты
8) Получение ответа от банковского сервиса (подтверждение или отмена транзакции с указанием сообщения об ошибке)

# 2.Перечень используемых инструментов с обоснованием выбора

1) Intellij IDEA - создание проекта, формирование отчетности
2) Docker - создание необходимых контейнеров с СУБД (MySQL, PostgreSQL, gate-simulator)
3) DevTools, Postman - тестирование API (отправка запросов, коды ответов)
4) DBeaver - работа с СУБД, SQL-запросами
5) GIT - создание удаленного репозитория, настройка CI

# 3.Перечень и описание возможных рисков при автоматизации

1) Недооценивание времени и усилий, затраченных на автоматизацию тестирования (большое кол-во багов, что влечет за собой много issues)
2) Не все проверки могут быть автоматизированы (например, проверка информации, сохраняемой в СУБД приложения)
3) Прогон автотестов на эмуляторе, а не на реальном банковском сервисе (могут быть неэффективны в реальности)

# 4.Интервальная оценка с учетом риска в часах

Среднее время на разработку и выполнение одного теста = 60 минут (*11 тестов)

Заведение баг-репорта на каждый дефект = 25 минут (*5 багов примерно)

Учет возможных дополнительных затрат времени и рисков (подготовка среды тестирования, подготовка сборок) = 300 минут

Другие непредвиденные проблемы = 400 минут

***Суммарная оценка = 25 часов***
